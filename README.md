# 🛠 AutoService — информационная система для сети автомастерских

## 📌 О проекте

**AutoService** — это веб-приложение для автоматизации федеральной сети автомастерских.  
Проект включает:

- **Backend** — REST API на FastAPI (Python) с PostgreSQL, JWT-аутентификацией и ролевой моделью (клиент, мастер, администратор).
- **Frontend** — одностраничное приложение на React + TypeScript с адаптивным дизайном (TailwindCSS), которое взаимодействует с API и предоставляет интерфейс для всех ролей.

Разработано в рамках курсовой работы по специальности «Информационные системы и программирование».

**Репозиторий:** [https://github.com/dcherep/praktika_cherep.git]

---

## ✨ Основные возможности

### Для клиентов (`client`)
- Регистрация, вход через JWT
- Создание / редактирование / просмотр своих заявок на ремонт
- Выбор мастерской и услуг из справочника
- Онлайн-оплата (заглушка, данные карт не хранятся)
- История заявок и отслеживание статуса (`new` → `in_progress` → `done`)

### Для мастеров смены (`master`)
- Просмотр заявок только своей мастерской
- Изменение статуса, назначение техников на заявку / конкретные услуги
- Создание заявки от имени клиента
- Удаление заявок в статусах `new` или `in_progress`
- Управление графиком работы и отпусками техников (своей мастерской)

### Для администраторов (`admin`)
- Полный доступ ко всем заявкам, пользователям, мастерским, услугам, городам
- Создание / блокировка учётных записей
- Управление справочниками (услуги, города, мастерские)
- Назначение любых техников на любые заявки

## 🆕 Новые функции (обновление от Черепенко Д.С.)

В рамках учебной практики были добавлены следующие улучшения фронтенд-части:

### 1. 📊 Сортировка услуг по цене
На странице услуг добавлены кнопки сортировки:
- 💰 От дешёвых к дорогим
- 💸 От дорогих к дешёвым

### 2. 💳 Статус оплаты заявки
В таблицу заявок добавлена отдельная колонка "Оплата" с выбором статуса:
- ❌ Не оплачено (красный)
- ⚠️ Частично (жёлтый)
- ✅ Оплачено (зелёный)

Статус синхронизируется через `localStorage` (в демонстрационных целях).

### 3. ⭐ Оценка работы мастера
Добавлена возможность оценить работу мастера звёздочками (от 1 до 5).  
Оценка доступна только после завершения заявки (статус "Готово") и сохраняется локально.

### 4. 💬 Чат с мастером
Внутри каждой заявки добавлен чат для общения клиента с мастером/администратором.  
Функционал:
- Отправка сообщений
- Визуальное разделение по ролям (👑 админ, 🔧 мастер, 👤 клиент)
- Сохранение истории чата в `localStorage`

### 5. 📅 Календарь записи
На странице создания заявки добавлен календарь для выбора удобной даты и времени:
- Доступны следующие 14 дней
- Выбор времени с 9:00 до 18:00
- Визуальное отображение занятых слотов
- Обязательный выбор времени перед оплатой (для клиентов)

## 🚀 Быстрый старт (Docker Compose)

### 1. Клонирование репозитория
```bash
git clone https://github.com/VANUSKA228/proect_new.git
cd proect_new
```

### 2. Настройка окружения (опционально)
Создайте файл `backend/.env` (можно скопировать из `.env.example`).  
Пример минимального содержимого:
```ini
DATABASE_URL=postgresql://postgres:secret@db:5432/autoservice
SECRET_KEY=your-very-secret-key-change-in-production
ACCESS_TOKEN_EXPIRE_MINUTES=480
CORS_ORIGINS='["http://localhost:3000","http://127.0.0.1:3000"]'
```

### 3. Запуск всех сервисов
```bash
docker compose up --build
```

После успешного запуска:

- **Backend API** → [http://localhost:8000](http://localhost:8000)
- **Документация API** (Swagger) → [http://localhost:8000/docs](http://localhost:8000/docs)
- **Frontend** → [http://localhost:3000](http://localhost:3000)

## 🔐 Аутентификация и роли

- **JWT stateless** (токен живёт 8 часов).
- Зависимость `get_current_user` в FastAPI проверяет токен и возвращает объект пользователя.
- `role_required('client', 'master', 'admin')` — декоратор для ограничения доступа.
- Пароли хэшируются **bcrypt** (cost=12).

**Матрица доступа (ключевые эндпоинты):**

| Эндпоинт                     | client | master | admin |
|------------------------------|--------|--------|-------|
| `POST /auth/login`           | ✅     | ✅     | ✅    |
| `POST /auth/register/client` | ✅     | ❌     | ❌    |
| `POST /auth/register`        | ❌     | ❌     | ✅    |
| `GET /orders/` (свои)        | ✅     | ❌     | ❌    |
| `GET /orders/` (мастерская)  | ❌     | ✅     | ❌    |
| `GET /orders/` (все)         | ❌     | ❌     | ✅    |
| `POST /orders/`              | ✅     | ✅     | ✅    |
| `DELETE /orders/{id}`        | ❌     | ✅*    | ✅    |
| `GET /users/`                | ❌     | ❌     | ✅    |
| `POST /services/`            | ❌     | ❌     | ✅    |
| `POST /payments/stub`        | ✅     | ❌     | ❌    |

\* Мастер может удалять только заявки в статусах `new` или `in_progress`.

## 🖥️ Фронтенд (подробнее)

- **Фреймворк**: React 18 + TypeScript (строгая типизация).
- **Сборщик**: Vite — быстрая разработка и оптимизированная production-сборка.
- **Маршрутизация**: React Router v6 (защищённые маршруты в зависимости от роли).
- **Управление состоянием**: Zustand (легковесный, без лишнего бойлерплейта).
- **Формы и валидация**: React Hook Form + Yup (схемы валидации).
- **HTTP-клиент**: Axios с перехватчиками для автоматической подстановки JWT-токена.
- **Стилизация**: TailwindCSS (утилитарные классы, адаптивная вёрстка).
- **Прокси в разработке**: Vite перенаправляет запросы с `/api` на `http://localhost:8000`, поэтому в коде фронта можно писать `/api/orders` и т.д.

**Страницы по ролям**:
- Публичные: `/login`, `/register`
- Клиент: `/dashboard`, `/orders`, `/orders/new`, `/orders/:id`, `/profile`
- Мастер: `/master/orders`, `/master/workers`, `/master/schedule`
- Администратор: `/admin/users`, `/admin/workshops`, `/admin/services`, `/admin/cities`

**Переменные окружения** (фронтенд):
- `VITE_API_URL` — базовый URL бэкенда (в `docker-compose.yml` задаётся как `http://localhost:8000`). В режиме разработки прокси работает независимо, но для продакшена nginx должен проксировать запросы на бэкенд или задавать полный URL.
